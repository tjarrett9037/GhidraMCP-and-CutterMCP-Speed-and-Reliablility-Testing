7) Controlled Patch Strategy on Local Copy (for analysis only)
====================================================================
Important note:
- Patches were applied ONLY to /home/kali/Lin8_copy (analysis copy), not to the original sample.

Patches applied:
1) fcn.00001f28 at 0x1f28:
   - Replaced prologue with: xor eax,eax ; ret
   - Effect: anti-debug check always returns "not debugged".
2) fcn.000016f2 at 0x16f2:
   - Replaced prologue with: xor eax,eax ; ret
   - Effect: process blacklist check always reports no analysis tools detected.
3) fcn.00001f7c signal/raise block:
   - NOP from 0x1f99 to 0x1fad (mov edi/call signal + mov edi/call raise sequence).
   - Effect: removes forced SIGTRAP generation.
4) fcn.0000154d branch at 0x1636:
   - NOP two-byte JNE so code always takes cipher path instead of memset-zeroing branch.
   - Reason: a runtime guard variable at 0x50a0 could force zeroization path; this ensured deterministic transform for extraction.

Why this was required:
- Without bypasses, debugger execution terminated before compare point.
- Needed stable execution to capture both memcmp operands.
