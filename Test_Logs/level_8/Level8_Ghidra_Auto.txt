Step 8 - Reverse transformation routine
---------------------------------------
Action:
- Decompiled `FUN_00401a89` and its internal block function `FUN_00401799`.

Raw data from `FUN_00401a89`:
- Uses constant string at 0x47800e:
  "@@expand 8-byte k"
- Builds 64-byte state from:
  - sigma words from "expand 8-byte k"
  - 64-bit key derived from literal 0x9988776655443322
  - 64-bit block counter
- For each 64-byte block:
  - Generates keystream via `FUN_00401799`
  - XORs with input bytes

Raw data from `FUN_00401799`:
- 20-round Salsa-like quarter-round schedule (10 loop iterations, each with column+row rounds).
- Adds original state words back at output (Salsa block function behavior).

Additional string confirmation:
- `list_strings(filter="expand")` returned:
  0047800e: "@@expand 8-byte k"

Interpretation:
- This is a Salsa20-family stream cipher setup with an 8-byte key variant.
- Because check does `transform(input) == ciphertext`, plaintext password can be recovered with:
  `plaintext = ciphertext XOR keystream`.

Why this mattered:
- Enabled deterministic offline recovery of original password bytes.
