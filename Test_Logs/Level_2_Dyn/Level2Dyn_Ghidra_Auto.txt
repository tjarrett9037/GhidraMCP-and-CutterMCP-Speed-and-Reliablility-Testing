================================================================
1) Investigation strategy
================================================================
I used a static reverse-engineering workflow:

1. Enumerate strings for obvious candidate secrets and user-facing output.
2. Enumerate imports to identify comparison/input-related APIs.
3. Enumerate exports/symbols to locate likely logic functions (e.g., main/check_*).
4. Decompile high-value functions and inspect control flow.
5. Confirm exact literal used in password comparison.

Reasoning:
- Strings often expose passwords, flags, status messages, and usage hints.
- Imports like strcmp indicate direct string comparison checks.
- Exported/symbol names can reveal function intent quickly in non-stripped binaries.
- Decompilation gives direct, high-confidence evidence of expected input.
